# arxiv-paper-fetch

Fetch paper metadata (title, author, date, abstract) from the arXiv API and present structured, human-readable summaries derived from the abstract.

## When to Use

> Use this skill when a user provides one or more arXiv paper IDs and asks for summaries, details, or an overview of the papers. Also use when the user asks to "fetch" or "look up" papers on arxiv. Do not use for searching arXiv by keyword or category; only for retrieving known paper IDs.

## Steps

1. **Validate paper IDs** — Confirm the user supplied valid arXiv paper IDs (e.g., `2606.16822`). If only the URL or partial identifier is given, extract the canonical ID. Ask the user for explicit IDs if none are found.
2. **Query the arXiv API** — Construct a GET request to `http://export.arxiv.org/api/query?id_list=<id1>,<id2>&max_results=<n>`. Use the programmatic interface (Python `urllib` or equivalent) to fetch XML results reliably. Set a reasonable timeout (e.g., 15 seconds).
3. **Parse the Atom XML response** — Extract each `<entry>` node. For each entry, capture:
   - Title (from `<title>`)
   - Authors (from `<author>/<name>`)
   - Published/updated date
   - PDF link (`<link title="pdf">`)
   - Abstract (from `<summary>`)
4. **Synthesize a structured summary for each paper** — Transform the raw abstract into a clear, scannable summary with the following elements:
   - Paper number (order in the list)
   - Emoji-coded type tag (e.g., `📄`)
   - Title, first author name, date, and PDF link
   - One-paragraph plain-language explanation of the core contribution
   - A "key insight" or "key dilemma" callout when the abstract implies one
5. **Present a cross-paper comparison table** — When multiple papers are fetched together, output a Markdown table that contrasts:
   - Type (e.g., theory/essay vs. system/tool)
   - A one-line summary of each paper
   - A joint perspective (how the papers complement or contrast each other) as a final paragraph
6. **Offer next actions** — End the summary with a prompt offering to deep-dive into one paper, search for related work, or adjust the search direction.

## Output

The final output is a Markdown report containing:

- A numbered paper summary for each fetched ID, following the structure described in Step 4.
- A comparison table and synthesis paragraph (when multiple papers are fetched).
- A closing line that offers the user options for next steps.
