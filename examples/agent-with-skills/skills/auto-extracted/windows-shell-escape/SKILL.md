# Windows Shell Escape

When a script or CLI tool is invoked from `bash` on Windows and arguments containing spaces, colons, or JSON special characters are truncated, use a temporary JSON file as a parameter bridge to completely avoid shell interpretation.

## Trigger Conditions

> The user provides a command that fails on Windows due to argument truncation, or needs to pass complex parameters (JSON, queries with AND/OR, colons, spaces) through `bash` to a script or tool. The script already supports a `--json` parameter or can be adapted to accept one.

## Steps

1. **Confirm the failure mode**: verify that arguments containing spaces, colons (`:`), or JSON braces are being truncated before they reach the target script. Check if the target script already supports a `--json` file parameter.
2. **If the script supports `--json` file path**: write the complex parameters into a temporary JSON file using `write_file` (e.g., `params.json`), then invoke the script with `--json params.json` instead of inline JSON strings.
3. **If the script does NOT support `--json`**: first modify the script to accept a `--json` parameter that reads from a file path. Add load logic similar to:
   ```python
   parser.add_argument("--json", type=str, default="", help="JSON file path for complex params")
   def load_from_json(path: str) -> dict:
       with open(path, "r", encoding="utf-8") as f:
           return json.load(f)
   ```
   Make JSON file parameters override equivalent CLI parameters when both are provided.
4. **Clean up**: after the script execution succeeds, remove the temporary JSON file to avoid leaving artifacts in the workspace.

## Output
