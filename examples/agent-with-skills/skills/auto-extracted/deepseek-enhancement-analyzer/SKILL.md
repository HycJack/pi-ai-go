# DeepSeek Enhancement Analyzer

Use this skill to systematically analyze a browser extension that enhances an AI web
chat interface (like DeepSeek) via `fetch` interception, and produce both an in-depth
architecture document and a tailored, streamlined design blueprint for a feature subset.

Primary success condition: generate two clear, actionable Markdown documents that
respect the original architecture while ruthlessly scoping a minimal viable version.

Follow the steps in order.

## 1. Audit core architecture and data flow

- Trace the main execution loop: `main-world script` → `fetch` intercept → `content script` → `background` → tool execution → result rendering.
- Identify every subsystem (Memory, Skill, MCP, tool calling, automation, sandbox, etc.).
- Map the technology stack used in each layer (framework, storage, UI, bundling).
- Document the critical design decisions: why `fetch` interception instead of API wrapping, why XML-based pseudo-tool-use, why local-first IndexedDB storage.

## 2. Produce the full architecture analysis document

- Write a detailed document covering: project overview, tech stack table, layered architecture diagram, complete 7-step data flow, and deep-dives on key subsystems (Memory, Skill, MCP, Tool Calling, Agent Loop, Automation).
- Include a file-tree overview and highlight the most reusable core modules.
- Clearly list the key design decisions and their rationale.

## 3. Define the streamlined feature scope

- Interview the user to confirm the exact feature subset they want (e.g., Knowledge Base, Memory, Skill System, Tool Calling).
- Explicitly list every feature from the original project that will be **removed**.
- Calculate estimated code reduction (lines of code and file count) based on the original project's structure.
- Confirm the "must-keep" core mechanism (`fetch` intercept + prompt injection + XML tool calls) remains intact.

## 4. Design the streamlined blueprint

- Define the new product positioning with a 1-sentence goal.
- For each retained system (Memory, Skill, Knowledge Base, Tools):
  - Design the minimal data structures.
  - Define the user interaction/trigger mechanism.
  - Outline the prompt injection and response parsing flow.
- Propose a lean technology stack (use the same stack as original unless there's a strong reason to change).
- Create a projected minimal file-tree (target ~40 files, ~3500 lines).
- Design a simple UI layout (e.g., 3-tab sidebar).
- Draft an 8-phase development plan with time estimates.

## 5. Finalize and output documents

- Write the final versions of:
  - `docs/项目架构深度分析.md` (Full Architecture Analysis)
  - `docs/精简版设计文档.md` (Streamlined Design Blueprint)
- Ensure both documents link to each other where relevant.
- Verify the streamlined blueprint is fully self-contained and can be implemented without referencing the original full project.

## Output

Deliver two Markdown documents saved to the `docs/` directory inside the project workspace:
1.  `docs/项目架构深度分析.md`: a comprehensive architecture analysis of the reference project.
2.  `docs/精简版设计文档.md`: a complete, actionable design blueprint for a minimized feature set.
